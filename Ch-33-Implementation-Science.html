    Chapter 33 — Implementation Science for School Change: The Gap Between Knowing and Doing Is an Engineering Problem (Hub)

    ◉ Thesis

    The most common failure in education is not choosing the wrong approach — it is failing to implement the right one. Schools attend to evidence, select promising practices, and then watch them dissolve within months because the organisational infrastructure to sustain change was never built. Implementation science argues that this is not a motivation problem, a leadership charisma problem, or a teacher quality problem. It is an engineering problem: known failure modes, predictable stages, measurable fidelity criteria, and structural supports that either exist or do not. Fixsen et al. (2005) demonstrated that identifying evidence-based practices is only half the challenge; reliable implementation is a separate discipline requiring its own methods, teams, and cycles of improvement. This hub chapter connects two sub-chapters that make the case from opposite ends — the first providing the frameworks and a 90-day blueprint for structured school change, the second arguing that implementation without rigorous evaluation is hope masquerading as strategy.

    Hub Overview

    This chapter is organized around two complementary sub-chapters:

    Frameworks and the 90-Day Blueprint

    Covers the foundational frameworks and practical timelines for implementation. This section addresses: - Why implementation science matters and the concept of implementation fidelity - Two key frameworks: NIRN's Active Implementation Frameworks and the EEF's staged model (Explore-Prepare-Deliver-Sustain) - A detailed 90-day implementation blueprint with four phases and specific outputs at each stage - Common implementation failure modes and how to avoid them - A worked example of implementing metacognitive instruction across a subject department

    Start here if you want to understand the stages of implementation and how to structure a 90-day cycle of change.

    Evaluation and Adaptive Design in Implementation

    Covers evaluation frameworks, measurement challenges, and how to design adaptive interventions. This section addresses: - The measurement problem: why competency assessment is genuinely difficult - Leading and lagging indicators: which metrics matter and when - Statistical approaches for tracking growth (multiple time points, realistic effect sizes) - Iterative evaluation using Plan-Do-Study-Act cycles - Proctor's framework for implementation, service, and student outcomes - Adaptive intervention design tailored to individual student response - The RE-AIM framework for comprehensive program evaluation - Application to the current course on core competencies - Connection to Danish school cultures and the dannelse tradition

    Start here if you want rigorous guidance on how to evaluate whether your implementation is working.

    Six Principles That Predict Whether Implementation Will Survive

    1. Implementation is a process, not an event. Schools move through stages of increasing competence, not a one-time launch.

    2. Fidelity matters more than adoption. High-fidelity implementation (delivering the core active ingredients consistently) predicts outcomes better than simply spreading a practice widely.

    3. Leading indicators predict outcomes. Track implementation fidelity and classroom behaviours during the launch phase; wait for outcome data only after the practice is stable.

    4. Middle-leader ownership is essential. Department heads, curriculum coordinators, and team leaders must understand and own the implementation work.

    5. Expect a multi-year journey. Fixsen et al. (2005) document that moving from initial implementation to full fidelity typically takes 2-4 years. Communicating this timeline prevents demoralisation.

    6. Evaluation serves improvement, not judgment. Use data to diagnose barriers and refine practice, not to blame teachers or schools.

    What This Means for Schools Using This Textbook

    This course on core competencies has embedded in it a 150-hour implementation. Teachers and leaders will use the course materials to understand what the competencies are, why they matter, and how to teach them. But understanding is not implementation. Schools using this course should:

    1. Use implementation science deliberately. Do not assume that distributing course materials results in competency teaching. Plan for the explore-prepare-deliver-sustain sequence outlined in Ch-33a-Frameworks-and-Blueprint.

    2. Form implementation teams that own the work and provide ongoing support.

    3. Select high-leverage practices from the course (e.g., think-alouds for metacognition, visible random groups for collaboration) and implement them well in one context before expanding.

    4. Track fidelity, not only outcomes. Use the practices as they are designed; observe lessons to ensure active ingredients are present.

    5. Celebrate and share early wins as the work progresses.
