    Appendix A4b — Exhibition Studio and Quality Guide: Developing Strong TOK Objects

    ◉ Overview

    This section walks students and teachers through the exhibition development process using the Studio Format. It includes a four-phase brainstorming and selection process, a quality checklist for evaluating strong object sets, common pitfalls and how to avoid them, and integration guidance with related resources.

    Exhibition Object Studio Format

    Brainstorming Phase (8 minutes for 8 objects)

    1. Choose one TOK prompt that genuinely interests you
    2. Generate eight candidate objects in 8 minutes (1 minute per object)
    3. One related to AI or technology
    4. One from your daily life
    5. One that shows different perspectives
    6. One about bias, ethics, or authority
    7. One from a subject you study
    8. One that contrasts with something else
    9. One that shows how knowledge changes
    10. One surprising or counterintuitive

    Don't evaluate yet; just generate.

    Evaluation Phase (Filter)

    For each candidate object, ask:

    • Is it specific and real? Not "science" but "a laboratory experiment on photosynthesis." Not "social media" but "a TikTok video about climate change I watched."
    • Does it raise a clear knowledge question? Could you write one sentence: "This object matters for TOK because..."?
    • Can a meaningful commentary be written? Will you have something substantive to say, or will you run out of ideas after one paragraph?

    Eliminate candidates that don't pass all three filters.

    Selection Phase

    From remaining candidates, select three that create contrast:

    • Don't use three versions of the same object (three AI outputs, three news articles)
    • Ensure they raise different kinds of knowledge questions
    • Ensure you have something different to say about each
    • Consider which objects you could find/access (they must be real)

    Justification Phase (1 sentence per object)

    Write one sentence for each: "This object matters for TOK because..."

    Examples: - "This object shows how AI can sound authoritative while making factual errors, raising the question of how we should evaluate sources when fluent language is abundant." - "This object reveals how the same historical event is interpreted differently depending on perspective, showing that history requires interpretation, not just collection of facts." - "This object demonstrates that bias in AI is not a flaw to be removed, but a reflection of choices embedded in data and algorithms, raising questions about whether objective algorithms are possible."

    Quality Checklist for Teachers and Students

    When evaluating a three-object exhibition set:

    Object relevance: - [ ] Each object is specific and real (not abstract or hypothetical) - [ ] The connection to a knowledge question is explicit - [ ] The student can explain why the object matters without the commentary

    Commentary quality: - [ ] The commentary addresses knowledge issues, not just the features of the object - [ ] Student voice and judgment are visible (not parroted definitions) - [ ] Different perspectives on the object are considered - [ ] Limitations or uncertainties are acknowledged - [ ] Ethical implications are addressed where relevant

    Set coherence: - [ ] The three objects create productive contrast (they don't all make the same point) - [ ] Together, they raise interesting questions about knowledge - [ ] The student has engaged in genuine inquiry, not just illustration of predetermined ideas

    Common Pitfalls and How to Avoid Them

    Pitfall: Objects that illustrate a concept but don't raise a question

    Wrong: Selecting a calculator to illustrate that "technology helps us do math."

    Right: Selecting a situation where a calculator gives a confident wrong answer, to raise the question: "What role does human judgment play when we use technology?"

    Pitfall: Commentaries that are description, not analysis

    Wrong: "This is a Wikipedia article. It was written by volunteers. It has been edited many times."

    Right: "This Wikipedia article shows how knowledge is collaborative and emergent—it changes as new information and perspectives are added. But does collaborative authorship make it more reliable or less? Compared to a peer-reviewed article, Wikipedia is more democratic but less systematically vetted. This raises the question: Is knowledge more reliable when it's vetted by experts, or when many people can contribute?"

    Pitfall: Objects disconnected from student's own inquiry

    Wrong: Selecting objects only from the "object bank" provided by the school.

    Right: Observing something in your own life, wondering about it, and developing that observation into an object. ("I noticed that my friends and I often cite AI summaries of articles we haven't read. I took a screenshot. This object raises a question for TOK...")

    Integration with Pedagogical Playbook

    This object bank complements the lesson sequences in A2-Pedagogical-Playbook, especially Lesson 5: Exhibition Workshop. Use this bank to show students what kinds of objects work, then have them develop their own using the studio format outlined above.

    The A2a-Six-Lesson-Mini-Sequence provides the full context for teaching exhibition development; Lesson 5 directly corresponds to the studio format here. The A2b-Protocols-Assessment-and-Troubleshooting section offers classroom protocols and assessment rubrics that apply to exhibition work.
